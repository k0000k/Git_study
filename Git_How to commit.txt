1. 변경 사항을 staging area에 기록하기.
staging area는 우리 컴퓨터에 있는 로컬 저장소와 git 저장소 사이에서 변경사항을 기록하는 공간이다.
변경사항이 생기면 add명령어로 staging area에 기록 할 수 있다. 
이 곳에 기록된 변경사항은 commit을 하기 전 까지 계속 변경이 가능하며 git 저장소에 영향을 주지 않는다.
정리하자면, 이곳에 모인 기록들은 commit을 하면 확정된다!

1-1. 변경사항 기록 (하나씩 할 때)
git add <파일명>

1-2. 변경 사항 기록 (현재 디렉토리에 모든 변경사항 기록)
git add .

3. 커밋추가
git commit -m "내용입력"

4. 변경사항 확인
git status
